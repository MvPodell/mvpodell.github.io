CSS Selectors Cheatsheet!

CSS selectors help HTML developers to specify styles for individual elements of their code. They can select HTML elements by type, class, ID, and attribute.

Selector Name Selector Format Level of Specificity Description
Universal * {...} Least All HTML elements are selected using the universal selector. When CSS rules are written in this format, every HTML on the page will be assigned those styling rules.
Type <type>{...} Low Selects HTML elements of a certain type and applies the same CSS rules to all of them. If a more specific selector addresses a certain element that would normally fall under this type's jurisdiction, HTML uses the rules laid down by the more specific CSS selector, therefore always overriding this selector since it is the weakest.
Class .className {...} Medium Classes are reusable labels for HTML elements. CSS styling rules inside a class are only applied to HTML elements with the specified class passed in as a class property. A class property can take in more than one class, separated by a space, to assign the CSS rules of more than 1 class to an HTML element.
ID #idName {...} Highest Selects the HTML element with the specificed id property passed in passed in. This can only be used once in an HTML file, and is the most specific selector.

Beyond these 4 selector types, there are a few other tricks that are useful when writing selectors.

Format Use
selector selector {...} Use this format in order to style elements that have ALL the specified elements.
selector, selector {...} Note the comma in this selector format. You can use this format to style multiple elements of different specifities (id, class, or type). This saves the coder the hassle of having to write out the same style rules for different selectors.
type.class {...} Use this format in order to style elements of a certain type that also have a specific class.
